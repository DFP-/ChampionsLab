## Architecture

### React Context (`src/lib/i18n/index.tsx`)

The i18n system is built on a React Context with a custom hook:

```tsx
const I18nContext = createContext<I18nContextValue | null>(null);

export function I18nProvider({ children, initialLocale = "en" }) { ... }
export function useI18n() { ... }
```

**Provider placement:** Wrapped around the entire app in `src/app/layout.tsx`.

**Initial locale detection:**
1. Cookie `cl-lang` (set on previous visit)
2. Falls back to `"en"`

**Locale persistence:**
- `localStorage.setItem("championslab-lang", locale)`
- Cookie: `cl-lang={locale};path=/;max-age=31536000;SameSite=Lax`
- `document.documentElement.lang` updated for SEO

### Translation Functions

The `useI18n()` hook provides these translators:

| Function | Purpose | Example |
|:---|:---|:---|
| `t(key, vars?)` | UI strings | `t("pokedex.title")` → "Pokédex" |
| `tp(name)` | Pokémon names | `tp("Venusaur")` → "Florizarre" (FR) |
| `tm(name)` | Move names | `tm("Tackle")` → "Charge" (FR) |
| `ta(name)` | Ability names | `ta("Overgrow")` → "Engrais" (FR) |
| `ti(name)` | Item names | `ti("Life Orb")` → "Orbe Vie" (FR) |
| `tn(name)` | Nature names | `tn("Adamant")` → "Hardi" (FR) |
| `ts(key)` | Stat labels | `ts("attack")` → "Atk" |
| `tt(type)` | Type abbreviations | `tt("fire")` → "FEU" (FR) |
| `tty(type)` | Type full names | `tty("fire")` → "Feu" (FR) |
| `tmd(move, fallback)` | Move descriptions | `tmd("Tackle", "...")` |
| `tad(ability, fallback)` | Ability descriptions | `tad("Overgrow", "...")` |
| `tid(item, fallback)` | Item descriptions | `tid("Life Orb", "...")` |

All translators fall back to English if a translation is missing.

## Translation Files

### UI Strings (`src/lib/i18n/{lang}.json`)
Nested JSON with dotted keys.

**Example (`en.json`):**
```json
{
  "nav": {
    "pokedex": "Pokédex",
    "teamBuilder": "Team Builder",
    "battleBot": "Battle Bot",
    "meta": "Meta Analysis",
    "tournaments": "Tournaments",
    "pokeSchool": "PokéSchool",
    "about": "About",
    "supportUs": "Support Us",
    "subtitle": "Pokémon Champions 2026"
  },
  "pokedex": {
    "title": "Pokédex",
    "description": "Browse {count} competition-legal Pokémon",
    "searchPlaceholder": "Search Pokémon, moves, abilities...",
    "sort": {
      "tier": "Tier",
      "name": "Name",
      "dex": "Dex #"
    }
  },
  "common": {
    "types": {
      "fire": "Fire",
      "water": "Water"
    },
    "stats": {
      "hp": "HP",
      "attack": "Atk"
    }
  }
}
```

**Usage:**
```tsx
const { t } = useI18n();
t("pokedex.title");                    // "Pokédex"
t("pokedex.description", { count: 147 }); // "Browse 147 competition-legal Pokémon"
```

## Adding a New Language

Follow these steps to add support for a new language (e.g., German `de`):

### 1. Create Translation Files

Copy English files as templates:

```bash
cp src/lib/i18n/en.json src/lib/i18n/de.json
cp src/lib/i18n/pokemon-names.fr.json src/lib/i18n/pokemon-names.de.json
cp src/lib/i18n/moves.fr.json src/lib/i18n/moves.de.json
cp src/lib/i18n/abilities.fr.json src/lib/i18n/abilities.de.json
cp src/lib/i18n/items.fr.json src/lib/i18n/items.de.json
cp src/lib/i18n/natures.fr.json src/lib/i18n/natures.de.json
cp src/lib/i18n/move-descriptions.en.json src/lib/i18n/move-descriptions.de.json
cp src/lib/i18n/ability-descriptions.en.json src/lib/i18n/ability-descriptions.de.json
cp src/lib/i18n/item-descriptions.en.json src/lib/i18n/item-descriptions.de.json
```

### 2. Translate All Keys

Edit each file, translating values while **preserving keys**.

**Important rules:**
- Keep variable placeholders like `{count}`, `{value}`, `{name}`
- Don't translate Pokémon names unless they have official regional translations
- Use professional competitive terminology
- Test text fits in UI elements

### 3. Register in i18n Context

Edit `src/lib/i18n/index.tsx`:

```tsx
// 1. Import the new UI dictionary
import de from "./de.json";
import pokemonNamesDe from "./pokemon-names.de.json";
import movesDe from "./moves.de.json";
import abilitiesDe from "./abilities.de.json";
import itemsDe from "./items.de.json";
import naturesDe from "./natures.de.json";
import moveDescsDe from "./move-descriptions.de.json";
import abilityDescsDe from "./ability-descriptions.de.json";
import itemDescsDe from "./item-descriptions.de.json";

// 2. Add to supported locales
export type Locale = "en" | "fr" | "es" | "it" | "de";  // Add "de"

// 3. Add to UI translations
const UI_TRANSLATIONS: Record<string, Dict> = { en, fr, es: es as Dict, it: it as Dict, de: de as Dict };

// 4. Add to game data dictionaries
const POKEMON_NAMES: Record<string, Record<string, string>> = {
  fr: pokemonNamesFr, es: pokemonNamesEs, it: pokemonNamesIt, de: pokemonNamesDe,
};
const MOVE_NAMES: Record<string, Record<string, string>> = {
  fr: movesFr, es: movesEs, it: movesIt, de: movesDe,
};
// ... etc for abilities, items, natures, descriptions
```

### 4. Add Language Selector Entry

Edit `src/components/language-selector.tsx` to add the new language option with a flag icon.

### 5. Test Thoroughly

- Check all pages render correctly
- Verify no text overflow in buttons/cards
- Test search functionality (Pokémon names should still work in English)
- Check RTL if applicable (not currently supported)

### 6. Update Documentation

Update `I18N.md` and `CONTRIBUTING.md` to list the new language.

## Translation Guidelines

### Do Translate
- UI labels, buttons, descriptions
- Move names (use official translations)
- Ability names (use official translations)
- Item names (use official translations)
- Nature names (use official translations)
- Strategy tips and educational content

### Don't Translate
- Pokémon names (use official regional names only)
- Variable placeholders: `{count}`, `{name}`, `{value}`
- Technical terms that are universally understood: "ELO", "BST", "OHKO"
- URLs and email addresses

### Terminology Consistency

## Maintenance Tips

- When adding new UI strings, add them to **all** language files
- Use `❌` markers in comments to indicate untranslated keys
- Run the app in each language periodically to catch overflow issues
- Keep game data translations (moves/abilities/items) synced with official game releases
